KEY SCHEMA FEATURES IMPLEMENTED

CRITICAL DESIGN INSIGHT #1: Legal Status ≠ Access Pathway
   ✅ Two separate enum fields
   ✅ Example: Psilocybin is legal_status=PROHIBITED but access_pathway=RIGHT_TO_TRY
   ✅ This is the schema's core insight - verified with multiple examples

CRITICAL DESIGN INSIGHT #2: Arbitrage as Derived View
   ✅ total_access_cost_usd computed from: procedure_price + travel_cost + accommodation
   ✅ No stored "arbitrage" field - computed at query time
   ✅ TAL (therapeutic adequacy layer) falls out as special case

CRITICAL DESIGN INSIGHT #3: Jurisdiction Nesting
   ✅ parent_id + level enum supports federal→state→city hierarchy
   ✅ Example: US Federal (sovereign) → Oregon (subnational) → municipalities
   ✅ Example: Honduras (sovereign) → Próspera ZEDE (special_zone)
   ✅ Handles federated systems (legal at state, illegal federally) correctly

CRITICAL DESIGN INSIGHT #4: Evidence Lives on Procedure-Condition Pair
   ✅ ProcedureIndication table stores E1-E8 evidence grades
   ✅ Not on AccessRecord (which is procedure × jurisdiction join)
   ✅ Same procedure has same evidence globally; only legal/price/oversight differ

CRITICAL DESIGN INSIGHT #5: Volatility Tracking Prevents Map Decay
   ✅ volatility enum: STABLE / PENDING_LEGISLATION / ACTIVE_FLUX
   ✅ Guides re-verification priority - ACTIVE_FLUX cells need frequent checks
   ✅ Example: Oregon/Colorado psilocybin = PENDING_LEGISLATION (2025 launch risk)
